# Code Review Skill

When the user asks to review code, run a code review:

## 1. Run git diff and git status
- `git diff` - see all unstaged changes
- `git status` - see current state

## 2. Analyze Changes
Review all changes and identify:
- What files changed
- The nature of the changes (feat, fix, refactor, test, etc.)
- Any potential issues, bugs, or concerns
- Code quality and style consistency

## 3. Report
Present findings to the user:
- Summary of changes
- Any issues found
- Recommendations for fixes (if any)

## 4. Fix Issues (if requested)
If the user asks to fix issues found during review:
- Make the necessary fixes
- Re-run review to verify
